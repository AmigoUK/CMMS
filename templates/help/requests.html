Understanding Requests

A request is a report of a problem. Once submitted, it moves through a lifecycle as the maintenance team reviews and resolves it.

{# ── Request Lifecycle ──────────────────────────────────────── #}
Request Lifecycle
{# Visual flow #}
New Acknowledged In Progress Resolved Closed
Status Meaning
New Just submitted, waiting for review
Acknowledged Supervisor has confirmed it's valid
In Progress A work order has been created
Resolved Work is done, issue fixed
Closed Fully closed, no further action
Cancelled Request was cancelled
{# ── Tracking Your Requests ─────────────────────────────────── #}
Tracking Your Requests

After submitting, you can track your request from the Dashboard or the Requests page.

Click on any request to see its full details, activity timeline, and linked work order.

{# ── Activity Timeline ──────────────────────────────────────── #}
Activity Timeline

Every request has an activity timeline showing what happened and when:

  • Kasia submitted this request — 3 Apr 10:15
  • Jan acknowledged the request — 3 Apr 14:30
  • Jan converted to Work Order WO-20260403-001 — 4 Apr 09:00
  • Kasia: "It's getting worse, now leaking" — 4 Apr 11:20
{# ── Adding Comments ────────────────────────────────────────── #}
Adding Comments

You can add comments to your request at any time (unless it's closed or cancelled).

Use comments to provide updates, clarify details, or ask questions.

Supervisors can also comment — check back for responses.

{# ── Uploading Files ────────────────────────────────────────── #}
Uploading Files

You can upload photos and documents to your request.

Supported formats: JPEG, PNG, WebP, GIF, PDF, Word, Excel.

Photos help technicians understand the problem before arriving on site.

Admin Guide

This guide covers system administration — managing users, teams, sites, and settings.

{# ── Managing Users ─────────────────────────────────────────── #}
Managing Users

User accounts control who can access the system and what they can do.

  • Create a user: Go to Admin → Users → New User. Enter their name, email, and assign a role and site(s).
  • Edit a user: Click any user in the list to update their details, role, or site assignments.
  • Toggle active/inactive: Deactivate a user to revoke access without deleting their account. Reactivate at any time.
  • Reset password: Admins can reset a user's password from the user edit page.
Roles
Role Description
Admin Full system access including user management, settings, and all sites.
Supervisor Can triage requests, create and assign work orders, and manage property and parts.
Technician Can view and execute assigned work orders, log parts and time.
Reporter Can submit maintenance requests and track their progress.
{# ── Managing Teams ──────────────────────────────────────────── #}
Managing Teams

Teams group technicians together for work assignment and scheduling.

  • Internal teams — Your own maintenance staff, organised by trade or area (e.g. Electrical, Mechanical, Facilities).
  • Contractor teams — External service providers who handle specialist or overflow work.
  • Creating a team: Go to Admin → Teams → New Team. Enter a name, select the type (internal/contractor), and add members.
{# ── Managing Sites ──────────────────────────────────────────── #}
Managing Sites

Sites represent the physical locations your organisation maintains.

  • Adding a site: Go to Admin → Sites → New Site. Enter the site name, address, and a short site code.
  • Site codes — Short identifiers (e.g. HQ, DEPOT-01) used in reports and labels for quick reference.
  • Users and property items are assigned to specific sites, so data stays organised per location.
{# ── Settings ────────────────────────────────────────────────── #}
General Settings

System-wide configuration options available to administrators.

  • Anonymous reporting — Toggle whether visitors can submit maintenance requests without signing in. When enabled, anyone with a QR code link can report a problem.
  • Name/email requirements — Configure whether anonymous reporters must provide their name and/or email address when submitting a request.
{# ── SMTP / Email Settings ───────────────────────────────── #}
SMTP / Email Settings

Configure outgoing email so the system can send notifications and reports.

  • SMTP Host — The mail server address (e.g. smtp.gmail.com, mail.yourdomain.com).
  • SMTP Port — Common ports: 587 (STARTTLS), 465 (SSL/TLS), 25 (unencrypted, not recommended).
  • TLS/SSL — Enable TLS for encrypted connections. Recommended for all production setups.
  • Username & Password — Credentials for SMTP authentication.
  • From address — The sender email address that recipients will see.
Test Connection — After entering your SMTP settings, click the Test Connection button to send a test email and verify everything works before saving.
{# ── PM Settings ─────────────────────────────────────────── #}
PM Settings

Configure how preventive maintenance (PM) tasks generate work orders.

Setting Description
Auto-generate days How many days before the due date the system should automatically create a work order for a PM task.
Lead days The advance notice period used when scheduling PM work orders, allowing time for planning and parts procurement.
Overdue thresholds Number of days past due before a PM task is flagged as overdue. Can be configured with warning and critical levels.
WO prefix A custom prefix for PM-generated work order numbers (e.g. PM-) to distinguish them from manually created work orders.
Navigate to Admin → PM Settings to configure these values.
{# ── Custom Fields ───────────────────────────────────────── #}
Custom Fields

Define custom data fields for property items, configured independently per site.

  • Per-site configuration — Each site can have a different set of custom fields. Go to Admin → Custom Fields and select a site to configure its fields.
  • Up to 5 fields per site — You can define a maximum of 5 custom fields for each site.
  • Available field types:
    • text — Single-line text input.
    • long_text — Multi-line text area.
    • date — Date picker. Date fields are tracked by the Expiry Report.
    • image — Image upload field.
  • Field label — Give each field a descriptive label that will be shown on the property form and detail page (e.g. "Warranty Expiry", "Certification Date", "Nameplate Photo").
{# ── Permissions Matrix ──────────────────────────────────── #}
Permissions Matrix

Fine-grained access control beyond the standard role system.

  • Role-based defaults — Each role (Admin, Supervisor, Technician, etc.) comes with a default set of CRUD permissions (Create, Read, Update, Delete) for each module.
  • Per-user overrides — You can grant or revoke individual permissions for specific users, overriding their role defaults. Go to Admin → Permissions to manage overrides.
  • Module-level control — Permissions are organised by module: Requests, Work Orders, Property, Parts, Reports, etc.
Changes to permissions take effect immediately upon save. The user will see the updated access on their next page load.
{# ── Translation Management ──────────────────────────────── #}
Translation Management

Manage the text strings used throughout the interface in each supported language.

  • Edit strings — Go to Admin → Translations to view and edit translation strings for each language. You can search for specific keys or filter by untranslated entries.
  • Export CSV — Export translation data as a CSV file for offline editing or handoff to a translator. Re-import after editing.
  • Language settings — Configure which languages are available and set the default language for the system under Admin → Language Settings.
{# ── Help Content Editor ─────────────────────────────────── #}
Help Content Editor

Administrators can edit the help pages displayed in the Help Centre directly from the admin panel.

  • Navigate to Admin → Help Content to see all help pages.
  • Select a page and language to edit its HTML content in a rich text editor.
  • Changes are saved to the database and take effect immediately.
  • English help pages are served from templates; translated pages (e.g. Polish) are served from the database and can be edited here.

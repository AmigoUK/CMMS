Reports

The Reports module turns the operational data your team enters daily — work orders, part usage, time logs, transfers — into spend-by-site insights with date filters and CSV export.

Who can see Reports? Supervisors and admins. The Reports nav item is hidden for users, contractors, and technicians.
{# ── What's in the Reports section ───────────────────────── #}
What's available
  • Spend Overview — parts consumed + labor cost + transfers in/out, broken down per site, for any time window. CSV export.
  • Parts to Order (linked from /parts/reorder) — low-stock list per site with cross-site surplus suggestions and pending-inbound netting.
  • Transfers ledger — every transfer with status (pending/completed/cancelled) and value.
{# ── Time-frame filters ────────────────────────────────────── #}
Time-frame filters

Every report uses the same filter row at the top. Pick a preset or enter a custom range, and tick "Compare" to see the previous equal-length window side-by-side.

  • Today / This week (Mon–Sun) / This month / This quarter / YTD
  • Last month / Last quarter
  • Custom — pick from / to dates manually
  • Compare — adds a column showing the same metrics for the previous window of equal length
{# ── Spend math ───────────────────────────────────────────── #}
How spend is calculated

Per site, per period, the total spend formula is:

net_spend = parts_consumed + labor + transfers_out − transfers_in
Component Source
Parts consumed PartUsage.quantity_used × unit_cost_at_use on work orders for the site, in range. Reversed usages are excluded.
Labor (duration_minutes / 60) × rate_at_log on time logs for the site, in range. Time logs without a rate snapshot count as zero.
Transfers out quantity × unit_cost_at_transfer for completed transfers from this site, in range — value left this site's budget.
Transfers in Same formula, transfers to this site — subtracted (inflow).

All component values are snapshotted at the time of the event (consumption, log creation, transfer request). Later edits to user rates or part costs do not rewrite history.

{# ── Labor cost setup ─────────────────────────────────────── #}
Labor cost setup

Labor cost only appears in spend reports for time logs that captured a rate. To enable it, an admin sets the Hourly Rate field on a user's profile (Admin → Users → Edit). All future time logs will snapshot that rate.

Existing historical time logs (created before a rate was set) show up as "rate not recorded" and contribute zero. There is no backfill — set the rate once and labor cost is correct going forward.

{# ── CSV export ────────────────────────────────────────────── #}
CSV export

Click Export CSV on any report and the same period filter is applied. The CSV has one row per site with columns: site_code, site_name, period, parts_consumed, labor, transfers_in, transfers_out, net_spend. Open in Excel, Google Sheets, or any accounting tool.

{# ── Common questions ──────────────────────────────────────── #}
Common Questions
Why does my labor total seem low?
Most likely time logs were created before a user had an hourly rate set. New time logs will count, old ones won't.
The "Transfers in" value is shown as negative — why?
From a single site's view, receiving a transfer is value arriving — it offsets that site's spend. It doesn't change the organisation total because the source site shows the same amount as Transfers out.
Can I see spend across all sites at once?
Admins see all sites in one table by default. Supervisors see the sites they have access to. Use the totals row at the top of the table for the cross-site sum.
How do I compare months?
Select This month (or Last month) and tick Compare. The previous equal-length window is shown next to each component so you can spot trends.
